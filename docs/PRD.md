# 產品需求文件 (PRD)：活動報名系統

## 1. 專案概述
- **背景與動機**：校園內有許多社團與系所舉辦活動，但現有的報名方式可能分散且難以管理。本系統針對此痛點，旨在打造一站式活動報名平台，讓使用者快速完成報名，同時協助主辦方有效管理參與名單與活動資訊。
- **目標用戶**：逢甲學生（活動參與者）、活動主辦方（例如系學會、社團等）。
- **核心價值主張**：提供一個直覺、整合的平台，降低學生進入活動的門檻，同時減輕主辦方管理名單及候補的作業負擔。

## 2. 功能需求
系統將提供以下主要功能：

1. **活動瀏覽功能**
   - *使用者故事*：作為一位逢甲學生，我希望能夠瀏覽目前所有開放報名的活動列表與詳細資訊，以便快速找到我感興趣的活動。
2. **線上報名功能**
   - *使用者故事*：作為一位逢甲學生，我希望能夠在系統上直接填寫資料並送出線上報名，以便確保我能成功卡位。
3. **活動建立與管理功能**
   - *使用者故事*：作為一位活動主辦方，我希望能夠建立新活動、設定活動內容與報名期限，以便順利發布活動資訊給全校學生。
4. **報名名單管理功能**
   - *使用者故事*：作為一位活動主辦方，我希望能即時查看並匯出所有報名者的名單資訊，以便進行後續的活動籌備與現場核對。
5. **報名狀態通知功能**
   - *使用者故事*：作為一位活動參與者，我希望在報名成功或遞補成功時，能收到系統的狀態通知，以便確認我的最終報名狀態。
6. **人數限制與候補機制**
   - *使用者故事*：作為一位活動主辦方，我希望系統能在名額滿時自動將後續報名者排入候補，並在有人取消時依序遞補，以便妥善控管參與人數。
7. **報名資料修改功能**
   - *使用者故事*：作為一位活動參與者，我希望在期限內可以隨時修改我的報名資料或取消報名，以便確保資料正確，並將名額釋出給候補的人。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Python + Flask
  - 模板引擎：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 系統須妥善處理併發報名的狀況，避免超賣名額的問題。
  - 妥善保護學生的個人資料，防範常見的安全漏洞（如 SQL Injection、XSS 攻擊）。

## 4. MVP 範圍

### Must Have (必備功能 - 初期實作)
- 活動建立與管理（限主辦方）。
- 活動列表瀏覽與資訊頁面。
- 使用者線上報名與防呆機制。
- 報名名單總覽與管理。

### Should Have (應有功能 - 中期實作)
- 人數限制控管與自動候補機制。
- 報名資料的修改與取消功能。

### Nice to Have (加分功能 - 未來擴充)
- 客製化報名狀態通知（Email 等）。
- 儀表板與數據分析（如報名進度追蹤）。
